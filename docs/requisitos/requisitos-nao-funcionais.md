# Requisitos Não Funcionais

| Código | Requisito Não Funcional |
|---|---|
| RNF01 | O sistema deverá possuir interface simples, intuitiva e acessível. |
| RNF02 | O sistema deverá ser responsivo para utilização em dispositivos web e mobile. |
| RNF03 | O sistema deverá proteger o acesso por meio de autenticação de usuário. |
| RNF04 | O sistema deverá armazenar os dados em banco PostgreSQL. |
| RNF05 | O sistema deverá apresentar bom desempenho nas consultas de receitas, despesas, perfis financeiros e relatórios. |
| RNF06 | O sistema deverá ser desenvolvido com Angular no frontend, Node.js no backend e Ionic na aplicação mobile. |
| RNF07 | O sistema deverá permitir manutenção e evolução futura do código-fonte. |
| RNF08 | O sistema deverá manter separadas as informações financeiras de cada perfil cadastrado pelo usuário. |
| RNF09 | O sistema deverá criptografar as senhas dos usuários armazenadas no banco de dados. |
| RNF10 | O sistema deverá validar o formato e autenticidade do CPF e CNPJ antes de persistir os dados. |
| RNF11 | O sistema deverá registrar logs de ações críticas como exclusões e alterações de dados. |
| RNF12 | O sistema deverá encerrar automaticamente a sessão do usuário após um período de inatividade. |
| RNF13 | O sistema deverá estar disponível em português brasileiro como idioma padrão. |
